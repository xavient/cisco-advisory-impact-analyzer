Cisco Advisory Impact Agent

Know which of your firewalls a Cisco advisory actually affects.

Paste a Cisco Event Response (ERP) link. The tool pulls every referenced advisory, reads its official product data, and uses AI to tell you exactly which ASA/FTD devices in your inventory are impacted — as a ready-to-share Excel report.

01Before you start

  • uv — Astral's Python tool installer (install once); it manages Python for you.
  • A FueliX API key — each person uses their own (see “Get your key” below).
  • Your firewall inventory — an Excel .xlsx (sheet FW_List) in the folder you run from.
  • Internet access — to sec.cloudapps.cisco.com and api.fuelix.ai.

02Install & run

  1. Install the tool with uv (one command, works the same on every OS): 
    uv tool install cisco-advisory-impact-analyzer \
  --from git+https://github.com/xavient/cisco-advisory-impact-analyzer
    This puts a single caia command on your PATH — no cloning, no virtual environment to activate.
  2. Configure your API key (stored once, per user; used from any folder): 
    caia --config
  3. Run it from the folder that holds your inventory .xlsx, then paste your Cisco ERP link when prompted: 
    cd /path/to/your/inventory-folder
caia
    The report analysis_output_<date>.xlsx appears in an output folder inside that same folder.
  4. Stay up to date — check your version and update via uv: 
    caia --version
caia --update
    A normal run also offers to update first when a newer version is available.
No uv? You can install with just Python 3.9+ (a venv + pip install . from the source) — see Install without uv in the README. The full walkthrough, output guide, and troubleshooting also live in the README.

03Get your FueliX API key

  1. Go to dev.fuelix.ai and log in.
  2. Open your Default project → the “…” menu → View.
  3. Under API keys, copy the key that’s already listed.
  4. Paste it when you run caia --config.

Keep your key private — it’s stored locally in a per-user config file (owner-only) and is never shared or committed.
