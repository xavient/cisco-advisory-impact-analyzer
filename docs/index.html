Cisco Advisory Impact Analyzer

Know which of your firewalls a Cisco advisory actually affects.

Paste a Cisco Event Response (ERP) link. The tool pulls every referenced advisory, reads its official product data, and uses AI to tell you exactly which ASA/FTD devices in your inventory are impacted — as a ready-to-share Excel report.

Discovers advisories

Reads an ERP page and finds every Cisco security advisory it references — no manual list-building.

AI impact analysis

Matches affected products and software releases against your inventory, flagging config-dependent cases as Indeterminate.

Excel report

01Before you start

  • Python 3.9 or newer — a one-time install (links in the repo README).
  • A FueliX API key — each person uses their own (see “Get your key” below).
  • Your firewall inventory — an Excel file named inventory.xlsx (sheet FW_List).
  • Internet access — to sec.cloudapps.cisco.com and api.fuelix.ai.

02Install & run

  1. Download & unzip. Grab the latest release and unzip it into a folder.
  2. Add your inventory. Copy your inventory.xlsx into that same folder, next to install.py.
  3. Open a terminal in the folder.
    Windows: Shift + right-click in the folder → “Open PowerShell window here”.  ·  macOS: open Terminal, type cd  then drag the folder in.  ·  Linux: right-click → “Open Terminal Here”.
  4. Run the installer (sets up everything and asks for your API key): 
    python3 install.py     # macOS / Linux
python install.py      # Windows  (or:  py install.py)
  5. Run it — then paste your Cisco ERP link when prompted: 
    python3 run.py         # macOS / Linux
python run.py          # Windows  (or:  py run.py)
    The report analysis_output_<date>.xlsx appears in the same folder.
The full walkthrough, output guide, and troubleshooting live in the repository README.

03Get your FueliX API key

  1. Go to dev.fuelix.ai and log in.
  2. Open your Default project → the “…” menu → View.
  3. Under API keys, copy the key that’s already listed.
  4. Paste it when the installer asks for FUELIX_API_KEY.

Keep your key private — it’s stored locally in a .env file and is never shared or committed.
