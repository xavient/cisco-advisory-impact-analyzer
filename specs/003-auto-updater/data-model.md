# Data Model: Self-Update Mechanism (Auto-Updater)

**Feature**: `003-auto-updater` | **Date**: 2026-07-17

This tool has no database. The "data model" is a small set of **on-disk artifacts** and the
**in-memory values** the updater derives from them, plus the **update state machine**. Field
types are conceptual; implementation is stdlib (`str`, `pathlib.Path`, `dict`).

---

## On-disk artifacts

### `VERSION` (tracked, shipped)
- **What**: The single source of truth for the installed version.
- **Location**: package/tool root.
- **Format**: one line, a semantic-style version string, no leading `v` (e.g. `1.2.0`).
- **Validation**: must parse as dot-separated integers (see *Semantic Version* below); a
  missing/unparseable file is treated as version *unknown* (behaves as "older" — offers
  update; never blocks `--version` from reporting "unknown").
- **Lifecycle**: stamped by the release workflow (= tag); overwritten by a successful update
  with the new package's `VERSION`; restored on rollback.

### `MANIFEST` (tracked, shipped)
- **What**: The list of paths contained in the release package (the "shipped set").
- **Location**: package/tool root.
- **Format**: newline-separated relative POSIX paths.
- **Validation**: non-empty; every listed path present in the package.
- **Lifecycle**: generated by the release workflow; replaced on update; used to compute
  removals (old manifest ∖ new manifest, minus preserve-list); restored on rollback.

### `.update-backup/<old-version>-<timestamp>/` (gitignored)
- **What**: A snapshot of the files an update overwrote/removed, plus the prior `VERSION` and
  `MANIFEST`.
- **Format**: mirrored relative paths under the timestamped folder.
- **Validation**: created before any file is modified; contains enough to fully restore the
  prior version (excluding the preserve-list, which is never touched).
- **Lifecycle**: created per update; newest is the rollback source; pruned to the last **2**
  on success (D8).

### `.update-in-progress` (gitignored, transient)
- **What**: A crash/interrupt recovery marker written just before the overlay and cleared on
  success or after auto-revert.
- **Format**: small key/value record — `old_version`, `target_version`, `backup_path`,
  `phase`.
- **Validation**: presence at startup ⇒ a prior update did not finish ⇒ recover (restore from
  `backup_path`).
- **Lifecycle**: write → (apply) → clear; survives a hard interrupt to drive recovery.

### `.env`, `inventory/`, `output/`, `.venv/` — the **Preserve-list**
- **What**: User/local data and environment. **Never** read, transmitted, backed up,
  overwritten, or deleted by the updater (FR-002, FR-017).
- **Invariant**: byte-for-byte unchanged across update and rollback (SC-002).

## In-memory / derived entities

### Semantic Version
- **Fields**: ordered integer components parsed from a version string (e.g. `1.2.0` →
  `(1, 2, 0)`); optional trailing components tolerated.
- **Rules**: comparison is numeric, component-wise, left to right; missing components treated
  as `0` (`1.2` == `1.2.0`); a leading `v` is stripped; an unparseable value sorts as
  *unknown* < any real version.
- **Operations**: `parse(str) -> tuple[int, ...]`; `compare(a, b) -> {equal | a_newer |
  b_newer}`.

### Installed Version
- **Source**: `VERSION` file (D1).
- **Fields**: `version: SemanticVersion | unknown`, `raw: str`.

### Latest Release
- **Source**: GitHub Releases API `tag_name`, or the `/releases/latest` redirect tag (D2).
- **Fields**: `tag: str`, `version: SemanticVersion`, `asset_url: str` (tag-pinned zip),
  `checksum_url: str`, `reachable: bool`.

### Release Package
- **Source**: the downloaded, verified `cisco-advisory-impact-analyzer.zip` (D3).
- **Fields**: `zip_path: Path`, `sha256_expected: str`, `sha256_actual: str`,
  `embedded_version: str`, `manifest: list[str]`, `root: Path` (dir containing `VERSION`,
  located per D11).
- **Validation (all must hold before apply)**: `sha256_actual == sha256_expected`; zip
  well-formed; `embedded_version == Latest.tag`.

### Update Check Result
- **Fields**: `status ∈ {up_to_date, update_available, ahead, latest_unknown}`,
  `installed: str`, `latest: str | None`.
- **Derivation**: `compare(installed, latest)` when reachable; `latest_unknown` otherwise.
- **Maps to**: FR-004/FR-005/FR-008; drives US1/US2/US4 messaging.

### Backup
- **Fields**: `path: Path`, `from_version: str`, `created_at: str (timestamp)`.
- **Use**: rollback source (FR-011) and auto-revert target (FR-012).

## Update state machine

```text
        ┌──────────────────────────────── check-only (FR-008) ──────────────────────────┐
        │                                                                                │
   [Idle] ──run──▶ [Checking] ──▶ compare(installed, latest)                             │
                        │              │           │            │                        │
                latest_unknown     up_to_date    ahead     update_available              │
                    │ (exit 3)      │(exit 0)   │(exit 0)         │                       │
                    ▼               ▼           ▼                 ▼                       │
                 report          "up to date" "ahead"      [Confirm] ──decline──▶ [Idle] │
                 & stop            & stop      & stop            │ (--yes skips)          │
                                                                 ▼                        │
                                                          [Downloading] ─fail─▶ (exit 4, no changes)
                                                                 ▼
                                                          [Verifying] ─fail─▶ (exit 4, no changes)
                                                                 ▼
                                                          [BackingUp]  (create backup; write marker)
                                                                 ▼
                                                    [Applying overlay + manifest deletions]
                                                                 │
                                              error ────────────┤
                                                 ▼               ▼
                                          [Auto-Revert]     [RefreshingDeps] (if requirements changed)
                                          restore backup          │
                                              │                   ▼
                                     revert ok │ revert fail   update VERSION/MANIFEST
                                     (exit 5)  │ (exit 6)      prune backups; clear marker
                                              ▼                   ▼
                                           [Idle-old]          [Done] (exit 0; report new version)

   Recovery: on any invocation, if [.update-in-progress] exists ⇒ restore from backup_path
             (drives FR-012 after a hard interrupt); --rollback restores the newest backup.
```

**Invariants across every transition**
- Preserve-list untouched at all times (FR-002).
- No installed file is modified until download **and** verification pass (FR-006).
- Between `BackingUp` and `Done`/`Auto-Revert`, the marker exists so an interrupt is
  recoverable — the install is never left in an unknown mixed state (FR-012).
- `.env` is never read or sent anywhere (FR-017).
